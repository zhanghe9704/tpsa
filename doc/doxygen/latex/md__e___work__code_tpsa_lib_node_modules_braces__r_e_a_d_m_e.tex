{\bfseries{Description}}\+: Strip backslashes that were used for escaping from the result.\hypertarget{md__e___work__code_tpsa_lib_node_modules_braces__r_e_a_d_m_e_autotoc_md696}{}\doxysection{What is \char`\"{}brace expansion\char`\"{}?}\label{md__e___work__code_tpsa_lib_node_modules_braces__r_e_a_d_m_e_autotoc_md696}
Brace expansion is a type of parameter expansion that was made popular by unix shells for generating lists of strings, as well as regex-\/like matching when used alongside wildcards (globs).

In addition to \char`\"{}expansion\char`\"{}, braces are also used for matching. In other words\+:


\begin{DoxyItemize}
\item \href{\#brace-expansion}{\texttt{ brace expansion}} is for generating new lists
\item \href{\#brace-matching}{\texttt{ brace matching}} is for filtering existing lists
\end{DoxyItemize}

$<$details$>$

There are two main types of brace expansion\+:


\begin{DoxyEnumerate}
\item {\bfseries{lists}}\+: which are defined using comma-\/separated values inside curly braces\+: {\ttfamily \{a,b,c\}}
\item {\bfseries{sequences}}\+: which are defined using a starting value and an ending value, separated by two dots\+: {\ttfamily a\{1..3\}b}. Optionally, a third argument may be passed to define a \char`\"{}step\char`\"{} or increment to use\+: {\ttfamily a\{1..100..10\}b}. These are also sometimes referred to as \char`\"{}ranges\char`\"{}.
\end{DoxyEnumerate}

Here are some example brace patterns to illustrate how they work\+:

{\bfseries{Sets}}


\begin{DoxyCode}{0}
\DoxyCodeLine{\{a,b,c\}       => a b c}
\DoxyCodeLine{\{a,b,c\}\{1,2\}  => a1 a2 b1 b2 c1 c2}

\end{DoxyCode}


{\bfseries{Sequences}}


\begin{DoxyCode}{0}
\DoxyCodeLine{\{1..9\}        => 1 2 3 4 5 6 7 8 9}
\DoxyCodeLine{\{4..-\/4\}       => 4 3 2 1 0 -\/1 -\/2 -\/3 -\/4}
\DoxyCodeLine{\{1..20..3\}    => 1 4 7 10 13 16 19}
\DoxyCodeLine{\{a..j\}        => a b c d e f g h i j}
\DoxyCodeLine{\{j..a\}        => j i h g f e d c b a}
\DoxyCodeLine{\{a..z..3\}     => a d g j m p s v y}

\end{DoxyCode}


{\bfseries{Combination}}

Sets and sequences can be mixed together or used along with any other strings.


\begin{DoxyCode}{0}
\DoxyCodeLine{\{a,b,c\}\{1..3\}   => a1 a2 a3 b1 b2 b3 c1 c2 c3}
\DoxyCodeLine{foo/\{a,b,c\}/bar => foo/a/bar foo/b/bar foo/c/bar}

\end{DoxyCode}


The fact that braces can be \char`\"{}expanded\char`\"{} from relatively simple patterns makes them ideal for quickly generating test fixtures, file paths, and similar use cases.\hypertarget{md__e___work__code_tpsa_lib_node_modules_braces__r_e_a_d_m_e_autotoc_md697}{}\doxysection{Brace matching}\label{md__e___work__code_tpsa_lib_node_modules_braces__r_e_a_d_m_e_autotoc_md697}
In addition to {\itshape expansion}, brace patterns are also useful for performing regular-\/expression-\/like matching.

For example, the pattern {\ttfamily foo/\{1..3\}/bar} would match any of following strings\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{foo/1/bar}
\DoxyCodeLine{foo/2/bar}
\DoxyCodeLine{foo/3/bar}

\end{DoxyCode}


But not\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{baz/1/qux}
\DoxyCodeLine{baz/2/qux}
\DoxyCodeLine{baz/3/qux}

\end{DoxyCode}


Braces can also be combined with \href{https://github.com/jonschlinkert/micromatch}{\texttt{ glob patterns}} to perform more advanced wildcard matching. For example, the pattern {\ttfamily $\ast$/\{1..3\}/$\ast$} would match any of following strings\+:

\hypertarget{md__e___work__code_tpsa_lib_node_modules_braces__r_e_a_d_m_e_autotoc_md698}{}\doxysection{Brace matching pitfalls}\label{md__e___work__code_tpsa_lib_node_modules_braces__r_e_a_d_m_e_autotoc_md698}
Although brace patterns offer a user-\/friendly way of matching ranges or sets of strings, there are also some major disadvantages and potential risks you should be aware of.\hypertarget{md__e___work__code_tpsa_lib_node_modules_braces__r_e_a_d_m_e_autotoc_md699}{}\doxysubsection{tldr}\label{md__e___work__code_tpsa_lib_node_modules_braces__r_e_a_d_m_e_autotoc_md699}
$\ast$$\ast$\char`\"{}brace bombs\char`\"{}$\ast$$\ast$


\begin{DoxyItemize}
\item brace expansion can eat up a huge amount of processing resources
\item as brace patterns increase {\itshape linearly in size}, the system resources required to expand the pattern increase exponentially
\item users can accidentally (or intentially) exhaust your system\textquotesingle{}s resources resulting in the equivalent of a DoS attack (bonus\+: no programming knowledge is required!)
\end{DoxyItemize}

For a more detailed explanation with examples, see the \href{\#geometric-complexity}{\texttt{ geometric complexity}} section.\hypertarget{md__e___work__code_tpsa_lib_node_modules_braces__r_e_a_d_m_e_autotoc_md700}{}\doxysubsection{The solution}\label{md__e___work__code_tpsa_lib_node_modules_braces__r_e_a_d_m_e_autotoc_md700}
Jump to the \href{\#performance}{\texttt{ performance section}} to see how Braces solves this problem in comparison to other libraries.\hypertarget{md__e___work__code_tpsa_lib_node_modules_braces__r_e_a_d_m_e_autotoc_md701}{}\doxysubsection{Geometric complexity}\label{md__e___work__code_tpsa_lib_node_modules_braces__r_e_a_d_m_e_autotoc_md701}
At minimum, brace patterns with sets limited to two elements have quadradic or {\ttfamily O(n$^\wedge$2)} complexity. But the complexity of the algorithm increases exponentially as the number of sets, {\itshape and elements per set}, increases, which is {\ttfamily O(n$^\wedge$c)}.

For example, the following sets demonstrate quadratic ({\ttfamily O(n$^\wedge$2)}) complexity\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{\{1,2\}\{3,4\}      => (2X2)    => 13 14 23 24}
\DoxyCodeLine{\{1,2\}\{3,4\}\{5,6\} => (2X2X2)  => 135 136 145 146 235 236 245 246}

\end{DoxyCode}


But add an element to a set, and we get a n-\/fold Cartesian product with {\ttfamily O(n$^\wedge$c)} complexity\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{\{1,2,3\}\{4,5,6\}\{7,8,9\} => (3X3X3) => 147 148 149 157 158 159 167 168 169 247 248 }
\DoxyCodeLine{                                    249 257 258 259 267 268 269 347 348 349 357 }
\DoxyCodeLine{                                    358 359 367 368 369}

\end{DoxyCode}


Now, imagine how this complexity grows given that each element is a n-\/tuple\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{\{1..100\}\{1..100\}         => (100X100)     => 10,000 elements (38.4 kB)}
\DoxyCodeLine{\{1..100\}\{1..100\}\{1..100\} => (100X100X100) => 1,000,000 elements (5.76 MB)}

\end{DoxyCode}


Although these examples are clearly contrived, they demonstrate how brace patterns can quickly grow out of control.

$<$/details$>$\hypertarget{md__e___work__code_tpsa_lib_node_modules_braces__r_e_a_d_m_e_autotoc_md702}{}\doxysection{Performance}\label{md__e___work__code_tpsa_lib_node_modules_braces__r_e_a_d_m_e_autotoc_md702}
Braces is not only screaming fast, it\textquotesingle{}s also more accurate the other brace expansion libraries.\hypertarget{md__e___work__code_tpsa_lib_node_modules_braces__r_e_a_d_m_e_autotoc_md703}{}\doxysubsection{Better algorithms}\label{md__e___work__code_tpsa_lib_node_modules_braces__r_e_a_d_m_e_autotoc_md703}
Fortunately there is a solution to the \href{\#brace-matching-pitfalls}{\texttt{ \char`\"{}brace bomb\char`\"{} problem}}\+: {\itshape don\textquotesingle{}t expand brace patterns into an array when they\textquotesingle{}re used for matching}.

Instead, convert the pattern into an optimized regular expression. This is easier said than done, and braces is the only library that does this currently.
