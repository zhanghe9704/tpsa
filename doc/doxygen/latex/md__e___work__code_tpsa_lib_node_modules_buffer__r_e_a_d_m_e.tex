\href{https://saucelabs.com/u/buffer}{\texttt{ }}

With \href{http://browserify.org}{\texttt{ browserify}}, simply `require(\textquotesingle{}buffer'){\ttfamily or use the}Buffer\`{} global and you will get this module.

The goal is to provide an API that is 100\% identical to \href{https://nodejs.org/api/buffer.html}{\texttt{ node\textquotesingle{}s Buffer API}}. Read the \href{https://nodejs.org/api/buffer.html}{\texttt{ official docs}} for the full list of properties, instance methods, and class methods that are supported.\hypertarget{md__e___work__code_tpsa_lib_node_modules_buffer__r_e_a_d_m_e_autotoc_md717}{}\doxysection{features}\label{md__e___work__code_tpsa_lib_node_modules_buffer__r_e_a_d_m_e_autotoc_md717}

\begin{DoxyItemize}
\item Manipulate binary data like a boss, in all browsers!
\item Super fast. Backed by Typed Arrays ({\ttfamily Uint8\+Array}/{\ttfamily Array\+Buffer}, not {\ttfamily Object})
\item Extremely small bundle size ({\bfseries{6.\+75\+KB minified + gzipped}}, 51.\+9\+KB with comments)
\item Excellent browser support (Chrome, Firefox, Edge, Safari 9+, IE 11, i\+OS 9+, Android, etc.)
\item Preserves Node API exactly, with one minor difference (see below)
\item Square-\/bracket {\ttfamily buf\mbox{[}4\mbox{]}} notation works!
\item Does not modify any browser prototypes or put anything on {\ttfamily window}
\item Comprehensive test suite (including all buffer tests from node.\+js core)
\end{DoxyItemize}\hypertarget{md__e___work__code_tpsa_lib_node_modules_buffer__r_e_a_d_m_e_autotoc_md718}{}\doxysection{install}\label{md__e___work__code_tpsa_lib_node_modules_buffer__r_e_a_d_m_e_autotoc_md718}
To use this module directly (without browserify), install it\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{npm install buffer}

\end{DoxyCode}


This module was previously called {\bfseries{native-\/buffer-\/browserify}}, but please use {\bfseries{buffer}} from now on.

If you do not use a bundler, you can use the \href{https://bundle.run/buffer}{\texttt{ standalone script}}.\hypertarget{md__e___work__code_tpsa_lib_node_modules_buffer__r_e_a_d_m_e_autotoc_md719}{}\doxysection{usage}\label{md__e___work__code_tpsa_lib_node_modules_buffer__r_e_a_d_m_e_autotoc_md719}
The module\textquotesingle{}s API is identical to node\textquotesingle{}s {\ttfamily Buffer} API. Read the \href{https://nodejs.org/api/buffer.html}{\texttt{ official docs}} for the full list of properties, instance methods, and class methods that are supported.

As mentioned above, `require(\textquotesingle{}buffer'){\ttfamily or use the}Buffer\`{} global with \href{http://browserify.org}{\texttt{ browserify}} and this module will automatically be included in your bundle. Almost any npm module will work in the browser, even if it assumes that the node {\ttfamily Buffer} API will be available.

To depend on this module explicitly (without browserify), require it like this\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{var Buffer = require('buffer/').Buffer  // note: the trailing slash is important!}

\end{DoxyCode}


To require this module explicitly, use `require(\textquotesingle{}buffer/'){\ttfamily which tells the node.\+js module lookup algorithm (also used by browserify) to use the $\ast$$\ast$npm module$\ast$$\ast$ named}buffer{\ttfamily  instead of the $\ast$$\ast$node.js core$\ast$$\ast$ module named}buffer\`{}!\hypertarget{md__e___work__code_tpsa_lib_node_modules_buffer__r_e_a_d_m_e_autotoc_md720}{}\doxysection{how does it work?}\label{md__e___work__code_tpsa_lib_node_modules_buffer__r_e_a_d_m_e_autotoc_md720}
The Buffer constructor returns instances of {\ttfamily Uint8\+Array} that have their prototype changed to {\ttfamily Buffer.\+prototype}. Furthermore, {\ttfamily Buffer} is a subclass of {\ttfamily Uint8\+Array}, so the returned instances will have all the node {\ttfamily Buffer} methods and the {\ttfamily Uint8\+Array} methods. Square bracket notation works as expected -- it returns a single octet.

The {\ttfamily Uint8\+Array} prototype remains unmodified.\hypertarget{md__e___work__code_tpsa_lib_node_modules_buffer__r_e_a_d_m_e_autotoc_md721}{}\doxysection{tracking the latest node api}\label{md__e___work__code_tpsa_lib_node_modules_buffer__r_e_a_d_m_e_autotoc_md721}
This module tracks the Buffer API in the latest (unstable) version of node.\+js. The Buffer API is considered {\bfseries{stable}} in the \href{https://nodejs.org/docs/latest/api/documentation.html\#documentation_stability_index}{\texttt{ node stability index}}, so it is unlikely that there will ever be breaking changes. Nonetheless, when/if the Buffer API changes in node, this module\textquotesingle{}s API will change accordingly.\hypertarget{md__e___work__code_tpsa_lib_node_modules_buffer__r_e_a_d_m_e_autotoc_md722}{}\doxysection{related packages}\label{md__e___work__code_tpsa_lib_node_modules_buffer__r_e_a_d_m_e_autotoc_md722}

Just use the buffer as a {\ttfamily Uint8\+Array}.\hypertarget{md__e___work__code_tpsa_lib_node_modules_buffer__r_e_a_d_m_e_autotoc_md726}{}\doxysubsection{convert blob to buffer}\label{md__e___work__code_tpsa_lib_node_modules_buffer__r_e_a_d_m_e_autotoc_md726}
