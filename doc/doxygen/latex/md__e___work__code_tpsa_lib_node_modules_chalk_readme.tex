Chalk comes with an easy to use composable API where you just chain and nest the styles you want.


\begin{DoxyCode}{0}
\DoxyCodeLine{const chalk = require('chalk');}
\DoxyCodeLine{const log = console.log;}
\DoxyCodeLine{}
\DoxyCodeLine{// Combine styled and normal strings}
\DoxyCodeLine{log(chalk.blue('Hello') + ' World' + chalk.red('!'));}
\DoxyCodeLine{}
\DoxyCodeLine{// Compose multiple styles using the chainable API}
\DoxyCodeLine{log(chalk.blue.bgRed.bold('Hello world!'));}
\DoxyCodeLine{}
\DoxyCodeLine{// Pass in multiple arguments}
\DoxyCodeLine{log(chalk.blue('Hello', 'World!', 'Foo', 'bar', 'biz', 'baz'));}
\DoxyCodeLine{}
\DoxyCodeLine{// Nest styles}
\DoxyCodeLine{log(chalk.red('Hello', chalk.underline.bgBlue('world') + '!'));}
\DoxyCodeLine{}
\DoxyCodeLine{// Nest styles of the same type even (color, underline, background)}
\DoxyCodeLine{log(chalk.green(}
\DoxyCodeLine{    'I am a green line ' +}
\DoxyCodeLine{    chalk.blue.underline.bold('with a blue substring') +}
\DoxyCodeLine{    ' that becomes green again!'}
\DoxyCodeLine{));}
\DoxyCodeLine{}
\DoxyCodeLine{// ES2015 template literal}
\DoxyCodeLine{log(`}
\DoxyCodeLine{CPU: \$\{chalk.red('90\%')\}}
\DoxyCodeLine{RAM: \$\{chalk.green('40\%')\}}
\DoxyCodeLine{DISK: \$\{chalk.yellow('70\%')\}}
\DoxyCodeLine{`);}
\DoxyCodeLine{}
\DoxyCodeLine{// ES2015 tagged template literal}
\DoxyCodeLine{log(chalk`}
\DoxyCodeLine{CPU: \{red \$\{cpu.totalPercent\}\%\}}
\DoxyCodeLine{RAM: \{green \$\{ram.used / ram.total * 100\}\%\}}
\DoxyCodeLine{DISK: \{rgb(255,131,0) \$\{disk.used / disk.total * 100\}\%\}}
\DoxyCodeLine{`);}
\DoxyCodeLine{}
\DoxyCodeLine{// Use RGB colors in terminal emulators that support it.}
\DoxyCodeLine{log(chalk.keyword('orange')('Yay for orange colored text!'));}
\DoxyCodeLine{log(chalk.rgb(123, 45, 67).underline('Underlined reddish color'));}
\DoxyCodeLine{log(chalk.hex('\#DEADED').bold('Bold gray!'));}

\hypertarget{md__e___work__code_tpsa_lib_node_modules_chalk_readme_autotoc_md756}{}\doxysubsubsection{chalk.$<$tt$>$$<$style$>$\mbox{[}.$<$style$>$...\mbox{]}(string, \mbox{[}string...\mbox{]})$<$/tt$>$}\label{md__e___work__code_tpsa_lib_node_modules_chalk_readme_autotoc_md756}
Example\+: `chalk.\+red.\+bold.\+underline(\textquotesingle{}Hello', \textquotesingle{}world\textquotesingle{});\`{}

Chain \href{\#styles}{\texttt{ styles}} and call the last one as a method with a string argument. Order doesn\textquotesingle{}t matter, and later styles take precedent in case of a conflict. This simply means that {\ttfamily chalk.\+red.\+yellow.\+green} is equivalent to {\ttfamily chalk.\+green}.

Multiple arguments will be separated by space.\hypertarget{md__e___work__code_tpsa_lib_node_modules_chalk_readme_autotoc_md757}{}\doxysubsubsection{chalk.\+enabled}\label{md__e___work__code_tpsa_lib_node_modules_chalk_readme_autotoc_md757}
Color support is automatically detected, as is the level (see {\ttfamily chalk.\+level}). However, if you\textquotesingle{}d like to simply enable/disable Chalk, you can do so via the {\ttfamily .enabled} property.

Chalk is enabled by default unless explicitly disabled via the constructor or {\ttfamily chalk.\+level} is {\ttfamily 0}.

If you need to change this in a reusable module, create a new instance\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{const ctx = new chalk.constructor(\{enabled: false\});}

\end{DoxyCode}
\hypertarget{md__e___work__code_tpsa_lib_node_modules_chalk_readme_autotoc_md758}{}\doxysubsubsection{chalk.\+level}\label{md__e___work__code_tpsa_lib_node_modules_chalk_readme_autotoc_md758}
Color support is automatically detected, but you can override it by setting the {\ttfamily level} property. You should however only do this in your own code as it applies globally to all Chalk consumers.

If you need to change this in a reusable module, create a new instance\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{const ctx = new chalk.constructor(\{level: 0\});}

\end{DoxyCode}


Levels are as follows\+:

0. All colors disabled
\begin{DoxyEnumerate}
\item Basic color support (16 colors)
\item 256 color support
\item Truecolor support (16 million colors)
\end{DoxyEnumerate}\hypertarget{md__e___work__code_tpsa_lib_node_modules_chalk_readme_autotoc_md759}{}\doxysubsubsection{chalk.\+supports\+Color}\label{md__e___work__code_tpsa_lib_node_modules_chalk_readme_autotoc_md759}
Detect whether the terminal \href{https://github.com/chalk/supports-color}{\texttt{ supports color}}. Used internally and handled for you, but exposed for convenience.

Can be overridden by the user with the flags {\ttfamily -\/-\/color} and {\ttfamily -\/-\/no-\/color}. For situations where using {\ttfamily -\/-\/color} is not possible, add the environment variable {\ttfamily FORCE\+\_\+\+COLOR=1} to forcefully enable color or {\ttfamily FORCE\+\_\+\+COLOR=0} to forcefully disable. The use of {\ttfamily FORCE\+\_\+\+COLOR} overrides all other color support checks.

Chalk can be used as a \href{http://exploringjs.com/es6/ch_template-literals.html\#_tagged-template-literals}{\texttt{ tagged template literal}}.


\begin{DoxyCode}{0}
\DoxyCodeLine{const chalk = require('chalk');}
\DoxyCodeLine{}
\DoxyCodeLine{const miles = 18;}
\DoxyCodeLine{const calculateFeet = miles => miles * 5280;}
\DoxyCodeLine{}
\DoxyCodeLine{console.log(chalk`}
\DoxyCodeLine{  There are \{bold 5280 feet\} in a mile.}
\DoxyCodeLine{  In \{bold \$\{miles\} miles\}, there are \{green.bold \$\{calculateFeet(miles)\} feet\}.}
\DoxyCodeLine{`);}

\end{DoxyCode}


Blocks are delimited by an opening curly brace ({\ttfamily \{}), a style, some content, and a closing curly brace ({\ttfamily \}}).

Template styles are chained exactly like normal Chalk styles. The following two statements are equivalent\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{console.log(chalk.bold.rgb(10, 100, 200)('Hello!'));}
\DoxyCodeLine{console.log(chalk`\{bold.rgb(10,100,200) Hello!\}`);}

\end{DoxyCode}


Note that function styles ({\ttfamily rgb()}, {\ttfamily hsl()}, {\ttfamily keyword()}, etc.) may not contain spaces between parameters.

All interpolated values ({\ttfamily chalk\`{}\$\{foo\}\`{}}) are converted to strings via the {\ttfamily .to\+String()} method. All curly braces ({\ttfamily \{} and {\ttfamily \}}) in interpolated value strings are escaped.\hypertarget{md__e___work__code_tpsa_lib_node_modules_chalk_readme_autotoc_md765}{}\doxysubsection{256 and Truecolor color support}\label{md__e___work__code_tpsa_lib_node_modules_chalk_readme_autotoc_md765}
Chalk supports 256 colors and \href{https://gist.github.com/XVilka/8346728}{\texttt{ Truecolor}} (16 million colors) on supported terminal apps.

Colors are downsampled from 16 million RGB values to an ANSI color format that is supported by the terminal emulator (or by specifying {\ttfamily \{level\+: n\}} as a Chalk option). For example, Chalk configured to run at level 1 (basic color support) will downsample an RGB value of \#\+FF0000 (red) to 31 (ANSI escape for red).

Examples\+:

\item `chalk.\+hex('\#\+DEADED\textquotesingle{}).underline(\textquotesingle{}Hello, world!\textquotesingle{}){\ttfamily  -\/}chalk.\+keyword(\textquotesingle{}orange\textquotesingle{})(\textquotesingle{}Some orange text\textquotesingle{}){\ttfamily  -\/}chalk.\+rgb(15, 100, 204).inverse(\textquotesingle{}Hello!\textquotesingle{})\`{}
\end{DoxyItemize}
